Python协程与异步编程

发表于 2016-08-26

背景知识

在现在流行的编程语言中，异步、协程、事件循环越来越受关注，本文试图探讨一下Python中协程原理，以及在python3.5中实现了IO多路复用、原生协程(coroutine)、与事件循环(event loop)的标准库asyncio

协程是什么？

根据维基百科：

Coroutines are computer program components that generalize subroutines for nonpreemptive multitasking, by allowing multiple entry points for suspending and resuming execution at certain locations.

协程是一种可以为非抢占式多任务生成子程序的计算机组件，它允许多个入口对程序进行挂起、继续执行等操作。与多线程和多进程相比，协程可以只利用一个线程就能更加轻便地实现多任务，将任务切换的开销降到最低,避免了无意义的调度。同时协程维护了正常的代码流程，在保证可读性的同时最大化地利用阻塞IO的空闲时间，成功避免了回调的难题。这里有个例子异步等待的 Python 协程很形象地解释了协程原理

yield与协程的关系

yield最初的引入是在Python2.2PEP 255,为了产生可迭代的序列来生成数据，并保持对生成器调用状态的跟踪。因此，我们利用yield关键字返回的同时，保存函数返回前的运行上下文以及下一条指令，需要的时候可以重新加载上下文从上次离开的下一条命令开始继续执行，我们成为这种return方式叫做yielding。试想如果两个函数f1()和f2()各自包含yield，主线程先启动f1(),当f1()执行到yield时，暂时返回主线程，主线程可以将执行权交给f2()，执行到f2()的yield后，可以再将执行权交给f1(),从而实现了在同一个线程中交错执行f1和f2，换言之，f1和f2是协同执行的程序，故名“协程”。

借助yield的“暂停”功能，在Python2.5PEP342中给生成器增加了新的方法，比如send()、throw()和close()，通过send()将协程从上次结束的地方继续执行，使得我们不仅可以暂停生成器，而且能够传递值到生成器暂停的地方，这与next()或for-in都能达到让生成器继续执行的效果。当然，next()也可以启动一个协程
下面我们通过一个简单的Producer－Consumer模型来解释yield创建协程：

def consume():
    while True:
        data = yield
        print('Consumed: {} '.format(str(data)))

def produce(consumer):
    import random
    while True:
        data = random.randint(0,10)
        print('Produced: {} '.format(str(data)))
        consumer.send(data)
        yield

consumer = consume()
next(consumer)
producer = produce(consumer)

for _ in range(10):
    next(producer)

从输出结果可以看出，任何时刻，只有一个协程在运行。

随着yield的进一步发展到Python3.3,在PEP 380中进一步增强了生成器，它通过yield from表达式，允许生成器将部分功能授予另一个生成器(即子生成器)。因此我们可以让生成器串联起来，使得返回值可以在调用栈中上下浮动，而不用对编码进行过多改动。

def bottom():
    return (yield 42)

def middle():
    return (yield from bottom())

def top():
    return (yield from middle())

gen = top()
vaule = next(gen)
print(value)   # 42
try:
    value = gen.send(vaule * 2)
except StopIteration as exc:
    value = exc.value
print(value)   # 84

总结： Python2.2中生成器可以让代码执行过程暂停，Python2.5可以传值给生成器暂停的地方，这使Python协程成为了可能。再加上Python3.3中的yield from，使串联生成器变得十分简单。

事件循环

根据维基百科，事件循环是一种等待程序分配事件或消息的编程架构。比如每个浏览器都存在JavaScript事件循环，当你点击某个按钮，这一动作将会发送给JavaScript事件循环，并检查是否注册过onclick回调来处理这一次点击。只要有注册过的回调函数就会伴随点击动作被执行。事件循环之所以是一种循环是因为它不停地收集事件并通过循环来应对这些事件。可以简单理解为“当A发生时，执行B”

asyncio

在Python中，我们可以通过asyncio来提供事件循环。asyncio重点解决网络服务中的问题。事件循环在这里将“来自套接字的I/O已经准备好读或写”当作“当A发生时”，事件循环对于释放Python GIL极为帮助。
asyncio要求所有用作协程的生成器必须由asyncio.coroutine装饰，并且可以对任何asyncio.Future对象使用yield from, 从而将其传递给事件循环，暂停当前协程的执行来等待某些事情的发生，future对象是asyncio的实现细节。一旦future对象获取了事件循环，它会一直在那里监听，直到完成它需要做的一切。当future完成自己的任务后，事件循环就会察觉到，暂停并等待在那里的协程会通过send()方法获取future对象的返回值并开始继续执行。 

import asyncio

@asyncio.coroutine
def countdown(number, n):
    while n > 0:
        print('T-times {} {}'.format(n, number))
        yield from asyncio.sleep(1)
        n -= 1

loop = asyncio.get_event_loop()
tasks = [
    asyncio.ensure_future(countdown("A",2)),
    asyncio.ensure_future(countdown("B",3))
]

loop.run_until_complete(asyncio.wait(tasks))
loop.close()

以上面代码为例。事件循环启动每一个countdown()协程，一直执行到每一个协程的yield from 和 acyncio.sleep()。这会返回一个asyncio.Future对象并将其传递给事件循环，并暂停当前协程的执行，事件循环会监控这一future对象，倒计时1秒后(同时也会检查其它正在监控的对象，比如其它协程)。1秒时间一到，事件循环将future对象的结果返回给协程，然后协程才能继续执行。直到事件循环被清空，所有协程执行完毕。

async/await

随着python3.5的出现，在PEP492中定义了async／await关键字来使用协程:

import asyncio

async def compute(x, y):
    print("Compute {} + {}".format(x, y))
    await asyncio.sleep(1)
    return x+y

async def print_sum(x, y):
    result = await compute(x, y)
    print("{} + {} = {}".format(x, y, result))

loop = asyncio.get_event_loop()
loop.run_until_complete(print_sum(1, 2))
loop.close()

需要注意的是，在async def定义的协程中，不能含有yield或yield from表达式，否则会报SyntaxError异常。我们可以将async/await看成python中异步编程的API，不等同于asyncio，它足够灵活可以并不依赖asyncio

Python多线程

发表于 2016-08-24

背景知识

并发与并行

  • Concurrency is about dealing with lots of things at once.

    如果一个系统支持有多个动作（Action）同时存在，那么就是一个并发系统。
    并发程序同时可以拥有多个线程，是计算机充分利用单核（一个CPU）实现（或者看起来）多个任务同时执行，其实并不是真的同时执行，因为在某个CPU时间段只有一个线程在运行，其它线程都属于挂起状态。

  • Parallelism is about doing lots of things at once.

    如果一个系统支持多个动作同时执行，那么就是一个并行系统。
    因此，只有多核才能实现真正的并行。

简言之，当我们聊起并发和并行，是指不同代码块交替执行的能力和不同代码块同时执行的能力。

线程与进程

在多任务系统中，每个应用程序都有一个自己的进程，操作系统会为进程分配一些执行资源，比如内存空间等，在进程中又可以创建一些线程，线程共享这些执行资源，并由操作系统调用。

线程状态

线程有多个状态：

  • New 创建
  • Runnable 就绪，等待调度
  • Running 运行
  • Blocked 阻塞
  • Dead 消亡

Python多线程

语言支持

  • thread(2.x)或_thread(3.x)是python多线程的底层支持模块，不建议使用。
  • threading是对thread进一步封装的标准库，推荐使用。

线程创建

我们通过覆盖threading.Thread类中的run方法来创建自己多线程类 

import time
import requests
from threading import Thread

class GetUrlThread(Thread):
    def __init__(self, url):
        self.url = url
        super().__init__()

    def run(self):
        resp = requests.get(self.url)
        print('{} request {} status_code is {}'.format(self.name, self.url, resp.status_code))

urls = [
      'http://www.baidu.com',
      'http://cn.bing.com',
      'http://www.sina.com.cn',
      'https://www.amazon.cn',
      'http://www.jd.com/'
    ]
threads = [GetUrlThread(url) for url in urls]

start = time.time()

for t in threads:
    t.start()

for t in threads:
    t.join()

print('Elapsed time: {}'.format(time.time()-start))

输出：

Thread-1 request http://www.baidu.com status_code is 200
Thread-2 request http://cn.bing.com status_code is 200
Thread-5 request http://www.jd.com/ status_code is 200
Thread-3 request http://www.sina.com.cn status_code is 200
Thread-4 request https://www.amazon.cn status_code is 200
Elapsed time: 2.341024160385132

解释：

  • 我们通过多线程来减少CPU的等待时间，当一个线程内等待网络请求返回时，CPU可以切换到其它线程进行网络请求
  • 线程是无序运行的
  • 每个线程必须执行run方法
  • 我们通过start()来告诉CPU执行线程中的run方法

线程合并

join()方法用于线程合并，主线程或某个函数创建了子线程，只要调用了子线程的join方法，那么主线程就会被子线程阻塞。上面的例子中，主线程将被每一个子线程阻塞，子线程之间无序运行。试想一下如果我们在每一个子线程start之后，就调用join方法 

for t in threads:
    t.start()
    t.join()

此时我们可以看到，每个子线程都被后来新建的子线程给阻塞了，因此线程将变得有序运行。

互斥锁与线程同步

多线程最常见的一个问题：数据共享。当多个线程都修改某个共享的数据时，使得线程运行结果不可预期，这种现象就是“线程不安全”
而线程同步能够保证多个线程安全访问竞争资源，最简单的同步机制是引入互斥锁。互斥锁给资源引入了状态：锁定或非锁定，当某个线程修改共享数据时，先将其锁定，其它线程不能更改，直到该线程释放资源，才将资源的状态变为“非锁定”。因此互斥锁保证了每次只有一个线程进行写操作，从而保证了多线程情况下数据的一致性。

import threading

class SharedCounter():
    def __init__(self, initial_value=0):
        self._value = initial_value
        self._value_lock = threading.Lock()

    def incr(self,delta=1):
        with self._value_lock:
            self._value += delta

    def decr(self, delta=1):
        with self._value_lock:
            self._value -= delta

counter = SharedCounter()
for t in [threading.Thread(target=counter.incr()) for i in range(50)]:
    t.start()
    t.join()
print(counter._value)

输出：50

上面例子中，我们通过Lock对象和with语句块保证互斥执行，每次只有一个线程可以执行with语句包含的代码块。with语句会在代码块执行前自动获得锁，并在执行结束后自动释放锁。当然你也可以使用acquire和release显式获得锁释放锁，不过with更加优雅安全。
当一个线程获得锁时，锁就进入“locked”状态，如果此时有另一个线程试图获得这个锁，该线程就会变成”blocked”状态，即“同步阻塞”。直到该线程释放锁，锁进入”unlocked”状态。线程调度从阻塞状态的线程中选择一个获得锁，并使得该线程进入运行（running）状态

死锁问题

死锁是每一个多线程程序都会面临的一个问题。根据经验，应该尽可能保证每一个线程都只能同时保持一个锁，一旦有线程同时申请多个锁，一切就不可预料了。死锁的检测和恢复，以及如何避免死锁是我们使用多线程的一个难题。建议参考Python3 cookbook来深入研究。

线程池

使用ThreadPoolExecutor轻松创建线程池

from concurrent.futures import ThreadPoolExecutor
import requests

def fetch_url(url):
    resp = requests.get(url)
    print('request {} status_code is {}'.format(url, resp.status_code))

pool = ThreadPoolExecutor(10)

for url in urls:
    pool.submit(fetch_url,url)

Python装饰器

发表于 2016-08-23

装饰器（Decorators）

装饰器(Decorators)是Python的一个重要部分。使用它，你可以在不改变原来函数的基础上轻易地扩展函数的功能，使你的代码更加Pythonic

函数也是对象

首先我们要明白，python中一切皆为对象，函数也是对象

In [1]: def hi(name='Bob'):
...:        return "hi " + name
...:

In [2]: print(hi())
hi Bob

In [3]: greet = hi   # 函数是对象，可以赋值给其它变量

In [4]: print(greet())
hi Bob

In [5]: del hi

In [6]: print(greet())
hi Bob

In [7]: print(hi())
NameError: name 'hi' is not defined

函数闭包

既然函数也是对象，那么我们就可以在函数中定义函数，从函数中返回函数：

def outer():
    x = 1
    y = 2
    def inner():
        print(x)
    return inner

foo = outer()

上面我们把inner作为一个函数被outer返回，并保存在foo变量中，不过foo()能正常运行吗？

从python的作用域规则来讲，x, y是函数outer的本地变量，而且只有当outer运行的时候才会存在。所以当我们在outer返回之后如果继续调用inner，此时变量x，y应该早已不复存在，可能会发生一个运行时候的错误。 

In [22]: foo()
1

万万没想到，返回的inner函数赋值给foo后仍能正常工作，这是为什么呢？
其实Python支持一个叫做“函数闭包”的特性，简言之，嵌套定义在非全局作用域里面的函数能够记住它在被定义的时候它所处的封闭命名空间。

In [21]: foo.__closure__
Out[21]: (<cell at 0x1062c31c8: int object at 0x1041f5450>,)

我们通过函数的__closure__属性可以看出，这个属性里面包含封闭作用域里的值，为什么只有一个int object呢？
因为封闭命名空间只会包含被捕捉到的值，比如x，虽然outer还定义了y，但是封闭作用域里面是不会有的。

下面我们稍微做一下变化： 

def outer(x):
    def inner():
        print(x)
    return inner
print1 = outer(1)
print2 = outer(2)
print1()
1
print2()
2

从上面的例子可以看出，我们通过闭包，即被函数记住的封闭作用域，轻松地创建了我们的自定义函数

装饰器也是闭包

其实装饰器就是一个闭包，只不过它把原函数当作参数，并返回一个装饰过的函数，从而在不修改原函数的基础上给原函数添加功能

def outer(func):
    def inner():
        print("before func")
        func()
        print("after func")
    return inner

def foo():
    print("i am foo")

decorated_foo = outer(foo)

In [32]: decorated_foo()

before func
i am foo
after func

由上面例子可以看出来，我们通过闭包，把foo函数当作参数传递给来装饰器，并返回了一个装饰过的foo函数，加强了foo的功能

@标识符

通常，我们使用@标识符来将装饰器应用到函数：

@outer
def foo():
    print("i am foo")

In [33]: foo.__name__
Out[33]: 'inner'

可是问题是，我们函数经过装饰之后__name__被inner改写了，Python提供来functools.wraps来解决这个问题

使用@wraps

from functools import wraps

def outer(func):
    @wraps(func)
    def inner():
        print("before func")
        func()
        print("after func")
    return inner

把原来的装饰器函数outer修改后，重新装饰foo函数

In [38]: foo.__name__
Out[38]: 'foo'

我们可以看出，此时被装饰之后的函数__name__并没有被装饰器修改

注意：@wraps接受一个函数来进行装饰，并加入了复制函数名称、注释文档、参数列表等等的功能。这可以让我们在装饰器里面访问在装饰之前的函数的属性，@wraps也是一个装饰器

使用args和kwargs

如果想进一步提升我们装饰器的功能，获取任意被装饰函数被调用时的参数，该怎么办呢？ 

def outer(func):
    @wraps(func)
    def inner(*args, **kwargs):
        print("Arguments are:{0}, {1}".format(args, kwargs))
        return func(*args, **kwargs)
    return inner

@outer
def foo(x, y=1):
    return x+y


In [46]: foo(1, 1)
Arguments are:(1, 1), {}
Out[46]: 2

In [47]: foo(1)
Arguments are:(1,), {}
Out[47]: 2

相当简单～

装饰器的典型应用场景

授权（Authorization）

装饰器能有助于检查某个人是否被授权去使用一个web应用的端点(endpoint)。它们被大量使用于Flask和Django web框架中。 

from functools import wraps

def requires_auth(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        auth = request.authorization
        if not auth or not check_auth(auth.username, auth.password):
            authenticate()  # 处理未授权的逻辑，比如返回HTTP 401
        return f(*args, **kwargs)
    return decorated

其实在比较常见的web框架比如，Flask中的路由注册，Django中的缓存机制，这些都是基于装饰器实现的

日志 (Log)

装饰器日志其实很常见啦，下面是一个简单的列子: 

from functools import wraps

def log(func):
    @wraps(func)
    def with_logging(*args, **kwargs):
        print(func.__name__ + " was called")
        return func(*args, **kwargs)
    return with_logging

@log
def addition_func(x):
    return x + x

Python生成器

发表于 2016-08-21

生成器（Generators）

生成器是用来创建Python序列的一个对象。使用它可以迭代庞大的序列，且不需要在内存中创建和存储整个序列。简言之，生成器是为迭代器产生数据的。

迭代器（iterators）

根据维基百科，迭代器是一个让程序员可以遍历一个容器（特别是列表）的对象。首先要理解几个核心概念：

  • 可迭代对象(Iterable)
    Python中任意的对象，只要它定义了可以返回一个迭代器的 __iter__ 方法，或者定义了可以支持下标索引的__getitem__方法，那么它就是一个可迭代对象，简言之，可迭代对象就是能提供迭代器的对象
  • 迭代器(Iterator)
    Python中任意的对象，只要定义了next(python2)方法或 __next__（python3），它就是一个迭代器
  • 迭代(Iteration)
    用简单的话讲，它就是从某个地方（比如一个列表）取出一个元素的过程。当我们使用一个循环来遍历某个东西时，这个过程本身就叫迭代

生成器是一种迭代器

所以，生成器本质是一种特殊的迭代器，但是你只能对其迭代一次。这是因为它们并没有把所有的值存在内存中，而是在运行时生成值。你通过遍历来使用它们，要么用一个“for”循环，要么将它们传递给任意可以进行迭代的函数和结构

例子

普通斐波那契函数

对于一个生成斐波那契数列的函数 

def fib(n):  
    a, b = 0, 1  
    result = []
    for i in range(n):  
         result.append(a)
         a, b = b, a+b  
    return result

上面函数会在内存中创建存储整个result序列，当输入参数n很大时，内存资源消耗很严重

通过定义类实现生成器

下面我们来定义一个Fib类，使其对象变成一个生成器 

class Fib():

    def __init__(self, n):
        self.n = n
        self.a, self.b = 0, 1

    def __iter__(self):
        return self

    def __next__(self):
        if self.n:
            now = self.a
            self.a, self.b = self.b, self.a + self.b
            self.n -= 1
            return now
        raise StopIteration()

虽然我们通过定义Fib类来实现了一个生成器，但是代码量明显增多，那么有没有更简洁优雅的方式实现一个生成器呢？

yield实现生成器

def fib(n):  
    a, b = 0, 1  
    for i in range(n):  
        yield a
        a, b = b, a+b

    print(i)

你可能会好奇，既然yield使fib函数返回了生成器对象，为什么所有值都被yield之后，没有触发StopIteration的异常？
答案很简单：for循环会自动捕捉到这个异常并停止调用next()

python中的可迭代对象

其实，Python中的list、set、str、tuple、dict、file都是iterable（可迭代对象）

比如对于str对象

s = 'abcdef'
next(s)

TypeError: str object is not an iterator

需要注意的是，str虽然是一个可迭代对象，但它并不是一个迭代器，我们可以通过内置函数iter来把一个可迭代对象返回一个迭代器对象

In [39]: s_iterator = iter(s)
In [40]: next(s_iterator)

Out[40]: 'a'
